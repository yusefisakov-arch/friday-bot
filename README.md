# FRIDAY Bot — Инструкция по деплою на Railway

## Переменные окружения (Environment Variables)
Добавьте в Railway:

| Переменная | Значение |
|-----------|---------|
| ANTHROPIC_API_KEY | ваш ключ от Anthropic |
| TELEGRAM_TOKEN | токен бота от BotFather |
| ALLOWED_USER_ID | ваш Telegram ID. Обязательно — без него бот не будет отвечать никому из соображений безопасности |
| DATABASE_URL | строка подключения к PostgreSQL (Railway добавляет автоматически при подключении плагина Postgres) |
| WEBAPP_URL | публичный https-адрес сервиса на Railway (см. ниже), без слэша на конце |

## Формы внутри Telegram (кнопки "📝 Задача", "💰 Финансы", "🤝 Договорённость", "🏠 Квартиры", "🔑 Заселение", "🚪 Выселение", "⚡ Коммуналка")
Каждая кнопка открывает мини-форму внутри Telegram (Web App):
- "📝 Задача" — описание, дедлайн, приоритет, исполнитель.
- "💰 Финансы" — тип (расход/доход), сумма, категория, комментарий.
- "🤝 Договорённость" — с кем, о чём договорились, дедлайн, следующий шаг.
- "🏠 Квартиры" — отдельная от личных финансов касса по сдаче квартир в субаренду: запись операций (приход/расход по квартире, категория, сумма, валюта MDL/EUR, дата, комментарий) или добавление новой квартиры в справочник (адрес, этаж, номер квартиры, wifi-логин/пароль, собственник и его контакты, сумма аренды собственнику, день аренды — постоянное число месяца для сбора показаний/оплаты). Кнопка "🏦 Касса квартир" показывает расчётный баланс и результат последней сверки. Сверку с фактической суммой на руках, произвольные отчёты и графики по операциям можно запросить прямо в чате.
- "🔑 Заселение" — при заезде нового квартиранта: выбрать квартиру, указать имя и фамилию, 1-2 телефона, дату заезда/выезда, сумму аренды и депозит текущего квартиранта, а также показания счётчиков на момент заселения (становятся новой базовой точкой для расчёта коммуналки).
- "🚪 Выселение" — при выезде квартиранта: выбрать квартиру, дату выезда, показания счётчиков на момент выезда (бот сразу считает итоговую коммуналку по разнице с прошлыми показаниями), сумму возврата депозита (записывается как расход по кассе квартиры) и комментарий, если часть депозита удержана. После сохранения данные текущего квартиранта (имя, телефоны, срок, аренда, депозит) очищаются — квартира готова к новому заселению.
- "⚡ Коммуналка" — выбрать квартиру и ввести новые показания счётчиков (свет, газ, вода, горячая вода, отопление). Бот вспоминает прошлые показания, считает разницу × тариф, прибавляет фиксированную часть и выводит итог; новые показания сохраняются как база для следующего раза. Галочка «Записать итог в кассу» сразу проводит сумму приходом по категории «Коммуналка».

## Список задач (кнопка "📋 Задачи")
Задачи сгруппированы по срокам: "Срочно (сегодня и просрочено)", "На завтра", "Позже", "Без срока". Внутри каждой группы задачи разбиты на подблоки по приоритету (🔴 высокий, 🟡 средний и ниже). У каждой задачи показан номер (#id) — по нему можно закрыть, изменить или удалить задачу прямо в чате ("закрой #123", "удали #124"), что удобно для одинаковых/дублирующихся задач. Дедлайн и исполнитель — в конце строки.

## Цели и планирование
FRIDAY помогает ставить цели на день/неделю/месяц и доводить их до результата. В чате: "поставь цель на неделю — ...", "покажи цели", "по цели #5 прогресс — сделал половину", "цель #5 достигнута". Бот проактивен: каждое утро показывает цели и помогает спланировать день; по понедельникам предлагает спланировать неделю и подвести итоги прошлой; 1-го числа — спланировать месяц. Большие цели предлагает разбить на задачи.

## Автоматические напоминания (каждый день в 8:00)
- **Утренний сбор** — открытые задачи на день.
- **SOP по квартирам** — фиксированные ежемесячные напоминания (фактуры, газ, Энергоком, интернет, оплата дома и т.д.), привязанные к числам месяца. Список — `get_sop_reminders`, добавить/убрать можно прямо в чате ("добавь напоминание на 5 число — ...", "убери напоминание про интернет").
- **Сбор показаний перед арендой** — если у квартиры указан "день аренды" (rent_day, задаётся в форме "Новая квартира" и не меняется при смене квартиранта), за день до этого числа каждый месяц бот напоминает собрать показания счётчиков и сделать просчёт перед встречей с квартирантом.
- **Окончание контракта** — в последние 10 дней перед lease_end (датой выезда текущего квартиранта, указывается при заселении) бот один раз напоминает спросить квартиранта про продление или выезд.

Чтобы бот начал следить за сроком контракта, при заезде нового квартиранта сообщите боту в чате: "квартирант на Лев Толстой 27 заехал с 15 июня по 15 декабря" — он сам сохранит даты через add_apartment (или используйте форму "🔑 Заселение").

Чтобы это заработало, нужно один раз настроить публичный адрес:
1. В Railway откройте сервис бота → вкладка Settings → Networking → Generate Domain.
2. Скопируйте выданный адрес (вида `https://xxxxx.up.railway.app`).
3. Добавьте переменную окружения `WEBAPP_URL` с этим адресом (без `/` на конце).
4. Перезапустите сервис.

Пока `WEBAPP_URL` не задан, кнопки работают по-старому — бот спрашивает поля по очереди.

## Расчёт коммуналки по счётчикам
Тарифы на свет/газ/воду (и при необходимости отопление/горячую воду) единые для всех квартир — задаются прямо в чате: "тариф на свет 2.5, на газ 18, на воду 30". Узнать текущие тарифы — "какие сейчас тарифы на коммуналку?".

Каждый месяц достаточно назвать новые показания счётчиков, например:
"Лев Толстой 27: свет 1350, газ 520, вода 45"

Бот сам помнит прошлые показания, считает разницу × тариф по каждой услуге, добавляет фиксированную часть квартиры (если задана, например интернет — указывается при добавлении/обновлении квартиры через `utilities_fixed`) и выводит разбивку с итоговой суммой. Если показание для квартиры/услуги вводится впервые — оно сохраняется как базовое, а стоимость в этот раз считается равной 0.

Разовые статьи — обслуживание дома по платёжке, отопление в "вторичках" по платёжке, уборка при выселении (от 500 до 1000) — называются каждый раз отдельно: "плюс обслуживание дома 200 и уборка 700". Они добавляются к итогу, но не сохраняются как постоянные.

После расчёта, если нужно записать итог в кассу квартир, попросите бота отдельно: "запиши это как коммуналку в кассу".

## Команды бота
- /start — приветствие
- /clear — очистить историю разговора
- /tasks — список открытых задач
- /decisions — список открытых договорённостей
- /finance — последние финансовые записи и итоги за месяц
- /memory — что бот запомнил о предпочтениях
- /selfdestruct — полностью стереть ВСЕ данные без возможности восстановления (требует подтверждения точной фразой)

Напишите боту @userinfobot в Telegram

## Структура кода
Код разбит на слои (зависимости направлены сверху вниз, без циклов):
- `core.py` — конфигурация, константы, клавиатуры, пул соединений с БД, общие утилиты (время МСК, Decimal, проверка пользователя).
- `db.py` — схема БД и все операции с таблицами.
- `ai.py` — системный промпт (с кэшированием), набор инструментов и обработка сообщения моделью, графики.
- `webapp.py` — HTTP-сервер мини-форм, API, проверка подписи Telegram, обработка отправленных форм.
- `scheduler.py` — брифинги, напоминания, ежедневные задачи по расписанию.
- `bot.py` — точка входа: телеграм-обработчики, диалоговая память, запуск всех сервисов.
