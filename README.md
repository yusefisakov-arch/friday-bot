# FRIDAY Bot — Инструкция по деплою на Railway

## Переменные окружения (Environment Variables)
Добавьте в Railway:

| Переменная | Значение |
|-----------|---------|
| ANTHROPIC_API_KEY | ваш ключ от Anthropic |
| TELEGRAM_TOKEN | токен бота от BotFather |
| ALLOWED_USER_ID | ваш Telegram ID. Обязательно — без него бот не будет отвечать никому из соображений безопасности |
| DATABASE_URL | строка подключения к PostgreSQL (Railway добавляет автоматически при подключении плагина Postgres) |
| WEBAPP_URL | публичный https-адрес сервиса на Railway (см. ниже), без слэша на конце |

## Формы внутри Telegram (кнопки "📝 Задача", "💰 Финансы", "🤝 Договорённость", "🏠 Квартиры")
Каждая кнопка открывает мини-форму внутри Telegram (Web App):
- "📝 Задача" — описание, дедлайн, приоритет, исполнитель.
- "💰 Финансы" — тип (расход/доход), сумма, категория, комментарий.
- "🤝 Договорённость" — с кем, о чём договорились, дедлайн, следующий шаг.
- "🏠 Квартиры" — отдельная от личных финансов касса по сдаче квартир в субаренду: запись операций (приход/расход по квартире, категория, сумма, валюта MDL/EUR, дата, комментарий) или добавление новой квартиры в справочник (адрес, собственник, аренда квартиранту/собственнику, депозит, срок аренды текущего квартиранта). Кнопка "🏦 Касса квартир" показывает расчётный баланс и результат последней сверки. Сверку с фактической суммой на руках, произвольные отчёты и графики по операциям можно запросить прямо в чате.

## Автоматические напоминания (каждый день в 8:00)
- **Утренний сбор** — открытые задачи на день.
- **SOP по квартирам** — фиксированные ежемесячные напоминания (фактуры, газ, Энергоком, интернет, оплата дома и т.д.), привязанные к числам месяца. Список — `get_sop_reminders`, добавить/убрать можно прямо в чате ("добавь напоминание на 5 число — ...", "убери напоминание про интернет").
- **Сбор показаний перед арендой** — если у квартиры указан срок аренды (lease_start/lease_end), за день до даты lease_start (по числу месяца, каждый месяц) бот напоминает собрать показания счётчиков и сделать просчёт перед встречей с квартирантом.
- **Окончание контракта** — в последние 10 дней перед lease_end бот один раз напоминает спросить квартиранта про продление или выезд.

Чтобы бот начал следить за сроком, при заезде нового квартиранта сообщите боту в чате: "квартирант на Лев Толстой 27 заехал с 15 июня по 15 декабря" — он сам сохранит даты через add_apartment.

Чтобы это заработало, нужно один раз настроить публичный адрес:
1. В Railway откройте сервис бота → вкладка Settings → Networking → Generate Domain.
2. Скопируйте выданный адрес (вида `https://xxxxx.up.railway.app`).
3. Добавьте переменную окружения `WEBAPP_URL` с этим адресом (без `/` на конце).
4. Перезапустите сервис.

Пока `WEBAPP_URL` не задан, кнопки работают по-старому — бот спрашивает поля по очереди.

## Расчёт коммуналки по счётчикам
Тарифы на свет/газ/воду (и при необходимости отопление/горячую воду) единые для всех квартир — задаются прямо в чате: "тариф на свет 2.5, на газ 18, на воду 30". Узнать текущие тарифы — "какие сейчас тарифы на коммуналку?".

Каждый месяц достаточно назвать новые показания счётчиков, например:
"Лев Толстой 27: свет 1350, газ 520, вода 45"

Бот сам помнит прошлые показания, считает разницу × тариф по каждой услуге, добавляет фиксированную часть квартиры (если задана, например интернет — указывается при добавлении/обновлении квартиры через `utilities_fixed`) и выводит разбивку с итоговой суммой. Если показание для квартиры/услуги вводится впервые — оно сохраняется как базовое, а стоимость в этот раз считается равной 0.

Разовые статьи — обслуживание дома по платёжке, отопление в "вторичках" по платёжке, уборка при выселении (от 500 до 1000) — называются каждый раз отдельно: "плюс обслуживание дома 200 и уборка 700". Они добавляются к итогу, но не сохраняются как постоянные.

После расчёта, если нужно записать итог в кассу квартир, попросите бота отдельно: "запиши это как коммуналку в кассу".

## Команды бота
- /start — приветствие
- /clear — очистить историю разговора
- /tasks — список открытых задач
- /decisions — список открытых договорённостей
- /finance — последние финансовые записи и итоги за месяц
- /memory — что бот запомнил о предпочтениях
